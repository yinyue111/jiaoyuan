# `jiaoyuan` 结构化评审 B

> 状态说明：这是一份历史评审记录。其核心建议大部分已被吸收进当前 `SKILL.md`，现在主要用于真实性复核、风险边界复查与后续扩写时的自检。

定位：从历史真实性与人物辨识度视角审阅当前 `SKILL.md` 与 `references/research/*.md`。

## 总评

这一版已经比初稿稳得多。

优点很明确：
- 主要矛盾、对象划分、阶段判断这些主轴都能在研究稿里找到根
- 工作流已经比一般人物 prompt 更像真正的 skill
- 边界意识也比初稿清楚，至少不再完全沉浸式冒充历史人物

但还有 5 个关键口子没完全收住：
- 还是有点把毛写成稳定一致的万能战略家，晚年断裂感不够
- 有些段落更像抽象组织顾问，不够像毛本人那套历史生成逻辑
- 群众路线被写得太顺，少了它后来被权力动员化的张力
- 外交现实主义已经写到，但“利用敌人之间矛盾”这一辨识度很高的招法还没单独立起来
- `SKILL.md` 里仍混着一些人物百科式段落，挤占了真正该留给执行规则的空间

下面给 5 条最高优先级修改建议。每条都附示例文本。

---

## 1. 不要把毛写成一个从 1927 到 1976 始终稳定一致的人

### 问题
当前版本虽然提到晚年错误，但整体阅读感还是“这是一套稳定正确的方法，偶尔用偏了”。
这不够像历史里的毛。

更准确的写法应该承认：
- 革命时期的毛
- 建国初制度重建时期的毛
- 1957 以后尤其大跃进到文革时期的毛

不是同一强度、同一边界、同一反馈机制。

### 修改建议
单独加入一个“时期分层与关键张力”小节，提醒 agent 不要把早期调查精神和晚年政治运动机械打通。

### 改后示例文本
```markdown
## 时期分层与关键张力

不要把毛泽东理解成一条平滑直线。
至少要分四段看：

- 1927—1945：革命与战争时期，强项是调查、根据地、统一战线、战略耐心
- 1949—1956：建国与制度重建时期，强项是国家建造，但高压治理同步增强
- 1957—1962：从人民内部矛盾到反右、大跃进，公开理论与实际处理出现强张力
- 1966—1976：文革与晚年时期，群众动员、路线斗争、个人权威和外交现实主义并存

回答涉及毛时，要先判断用户问的是哪一段，不要把某一时期的方法直接套到全部时期。
```

---

## 2. 把“群众路线与组织放大”改得更像毛，而不是通用组织学

### 问题
当前模型 5 有用，但写成了“把分散经验转成可执行力量”。这句话太像现代组织顾问。
毛的群众路线不是普通的用户洞察，也不是单纯社区运营。
它更接近：从群众中取材，再经过党与组织加工，重新回到群众中去执行。

### 修改建议
把模型名称和证据改得更贴近原始表述。
同时保留它后来可能滑向“代群众发言”的风险。

### 改后示例文本
```markdown
### 模型5 群众路线不是讨好群众，而是把群众经验变成组织路线
**一句话**：从群众中来，到群众中去。不是跟着群众情绪走，而是把分散经验提炼成路线，再组织群众执行。
**证据**：
- 农民问题始终是革命核心
- 《为人民服务》把政治正当性压到人民这一根线上
- 1948 年谈报纸工作时，反复强调干部和群众都得知道政策，才能形成一条心
**应用**：遇到复杂组织问题时，先看基层经验如何汇总，再看谁有能力把经验变成组织动作。
**局限**：一旦只剩“代表群众”而没有真实反馈回路，群众路线就会滑成动员口号，甚至反噬系统。
```

---

## 3. 单独写出“先定性质，再定方法”的危险与锋利

### 问题
现在 skill 已经写了对象划分和主要矛盾，但还没把毛特别有辨识度的一招写透。
那就是先给矛盾定性质，再决定处理方法。

这招非常毛。
也非常危险。
因为它一旦把本该是人民内部的问题改判成敌我问题，整个处理强度就会陡变。

### 修改建议
把这条写成高辨识度启发式，同时明确风险。

### 改后示例文本
```markdown
### 启发式 先定性质，再定方法
- 应用场景：组织争论、路线冲突、公共批评、统战关系
- 说明：先判断这是敌我矛盾，还是人民内部矛盾，再决定能否批评、能否联合、是否要斗争
- 案例：1957 年《关于正确处理人民内部矛盾的问题》提出了两分法，但同年从百花到反右的急转，说明这种定性边界并不稳定
- 风险：一旦定性升级，原本可讨论的问题会迅速变成高压处置问题
```

---

## 4. 把“利用敌人之间矛盾打开空间”单独立起来

### 问题
当前版本写了统一战线和尼克松访华，但还没把毛在大战略上极有辨识度的一手单独钉住。
这手不是泛泛的现实主义。
而是把原先的头号敌手临时变成次要矛盾，用来压更现实的威胁。

从西安事变到抗日统一战线，再到中美接近，这条线非常稳定。

### 修改建议
把它升级成独立启发式或补充模型说明。

### 改后示例文本
```markdown
### 启发式 利用敌人之间的矛盾打开空间
- 应用场景：多方博弈、弱势方破局、联盟重组、国际关系
- 案例：西安事变后的联蒋抗日，1971—1972 年借中美接近制衡苏联
- 说明：毛不是先问价值一致不一致，而是先问谁是当前更大的威胁，谁可以暂时转成可利用的力量
- 风险：这种做法高度依赖阶段判断，阶段判断一错，朋友敌人的排序就会失真
```

---

## 5. 继续给 `SKILL.md` 减肥，把人物百科和宣发尾巴彻底下沉

### 问题
当前版本虽然比初稿好多了，但 `身份卡`、`人物时间线`、`最新动态`、`智识谱系`、文末生成器署名这些内容，还是更像人物展示页，不像执行文件。

这会带来两个问题：
- 让 skill 继续膨胀
- 让 agent 更容易模仿人物介绍，而不是执行判断流程

### 修改建议
`SKILL.md` 只留真正要驱动回答的东西。
其余都移回 `references/research/`。

### 改后示例文本
```markdown
## 视角基底

这个 skill 重点提取毛泽东在四类问题上的判断方式：
- 主要矛盾
- 对象划分
- 阶段转换
- 统一战线与群众动员

不要把它当成人物传记。
要把它当成一种会带来风险和张力的分析框架。
```

---

## 最后结论

如果只再改一轮，我建议优先级是：

1. 先补“时期分层与关键张力”
2. 再把模型 5 改得更像真正的群众路线
3. 再把“先定性质，再定方法”单独钉住
4. 再把“利用敌人之间矛盾打开空间”单独钉住
5. 最后继续给 `SKILL.md` 减肥

一句话说完：
这版已经不虚了，但还差最后一口气。那口气不是更多材料，而是把毛身上的断裂、锋利和危险也一起写进去。
